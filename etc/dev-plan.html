JGraphT Development Plan


This is a working document where the development plan is maintained. The document is sorted according to versions in descending order (newest version at the top). For each version there is a list of development items related to that version. These items have one of the following statuses* attached to them:

proposed A proposed development item for a release, with name initials of who proposed it. After mail discussion, such item will either be committed, rejected, or deferred.
committed A item is one that we have decided to address for the release.
deferred A reasonable proposal that did not make it in to this release for some reason. It is marked as deferred with a brief note as to why they were deferred. Deferred plan items may resurface as proposed plan items at a later point.
rejected Plan items that were proposed but judged unworkable are marked as rejected plan items, with an accompanying summary of why they were dismissed. Keeping track of rejected items avoids repeating the discussion.
* these statuses were adopted from the Eclipse project plan.

Please try to use simple and tidy HTML for this document, to ease version compare.

 

Version 0.4.2

Min-cut max-flow algorithm. (proposed BN)

Shortest path algorithm. (proposed BN)

Minimum spanning tree algorithm. (proposed BN)

 

Version 0.4.1

Depth first iteration and breadth first iteration. (accepted)

A separate unit test hierarchy in 'testsrc' that shadows the package structure of 'src'. Package naming within 'testsrc' is the same as in 'src'. Class naming is the same as in 'src', only suffixed with 'Test'. (accepted)

Refactoring of inner classes of GraphFactory. (proposed LR)

Adding more rigorous unit tests for basic graphs. Basic graphs are the non-delegating graphs, which are based on the DefaultGraph class and are the very core of JGraphT. (proposed BN)

Graph connectivity predicates and iteration of connected components of undirected graphs. (proposed BN)

 

Version 0.4.0  (released)

Initial publicly available version of JGraphT.
